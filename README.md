# 收纳骑士 (Storage Knight)

**收纳骑士** 是一款结合了**背包管理（类似俄罗斯方块的拼图机制）**与**自动战斗**的策略 Roguelike 游戏。玩家需要通过在有限的背包空间内合理摆放各种形状的武器、防具和道具，激活强大的羁绊效果，从而击败不断变强的敌人。

## 🎮 核心玩法

### 1. 背包收纳与空间管理
- **拼图机制**：商店中出售的每一件物品都有不同的形状（如单格、2x2、L型等）。你需要将它们拖拽到背包网格中，尽可能利用每一寸空间。
- **背包扩充**：随着游戏的进行，你可以花费金币来扩充背包的行数和列数，容纳更多强大的装备。
- **物品合成**：将两个相同等级的同名物品拖拽到一起，即可合成更高等级的物品，不仅属性大幅提升，还能节省宝贵的背包空间。

### 2. 丰富的羁绊系统
物品之间不仅有基础属性，还带有各种强大的羁绊效果。合理搭配羁绊是通关的关键：
- **相邻触发 (Adjacent)**：物品必须上下左右相邻摆放才能触发（例如：剑与磨刀石相邻增加攻击力）。
- **对角触发 (Diagonal)**：物品必须在对角线位置相邻才能触发（例如：王冠在武器或防具的对角线提供暴击和生命加成）。
- **全局触发 (Global)**：只要物品存在于背包中即可触发，无需考虑摆放位置（例如：多件火焰法杖之间的共鸣）。

### 3. 冲突与负面羁绊
强大的力量往往伴随着代价。部分极品装备带有**负面羁绊（冲突）**。
- 例如：拥有极高基础攻击力的“诅咒之刃”，如果你的背包中同时存在“神圣”标签的物品，就会触发负面羁绊，大幅削弱你的攻击力。
- 在商店中，如果某件物品与你当前背包内的物品存在冲突，会显示红色的 **“⚠️ 冲突”** 提示。

### 4. 自动战斗与成长
- **回合制自动战斗**：整理好背包后，点击“开始战斗”，骑士将根据背包提供的总属性（生命、攻击、防御、暴击）自动与敌人交战。
- **金币循环**：击败敌人可以获得金币，用于在商店购买新物品、刷新商店或扩充背包。如果不需要某件物品，也可以将其拖出背包半价出售。

## ⚔️ 物品流派概览

- 🗡️ **近战流**：剑、长枪、磨刀石。简单直接的物理输出。
- 🛡️ **防战流**：盾牌、头盔。高额护甲，稳扎稳打。
- 🪄 **法师流**：法杖、药水。通过魔法物品和消耗品的联动获得高额属性。
- 🔥 **火焰流**：烈焰法杖系列。全局羁绊，数量越多威力越恐怖。
- ❄️ **冰霜流**：冰霜之盾系列。相邻羁绊，提供海量生命与防御。
- 🍖 **暴食流**：烤肉系列。提供巨额生命值上限。
- ✝️/🗡️ **神圣与黑暗**：互相排斥的两个流派。黑暗武器攻击力极高但排斥神圣；神圣物品在面对黑暗时能提供强大的防御加成。

## 🛠️ 技术栈与 Cocos Creator 迁移

本项目原生基于 **React 18 + Tailwind CSS** 开发。

### 🎮 Cocos Creator 3.x 移植指南
为了方便将核心业务逻辑移植到 **Cocos Creator 3.x** 游戏引擎中，我们在项目根目录下提供了 `cocos_scripts/` 文件夹。该文件夹包含了完全脱离 React 依赖的纯 TypeScript 核心逻辑，并封装为了 Cocos 的 `cc.Component`。

1. **获取代码**：将本项目中 `cocos_scripts/` 目录下的所有 `.ts` 文件复制到你的 Cocos Creator 项目的 `assets/Scripts/` 目录下。
2. **核心文件说明**：
   - `Items.ts`: 物品数据定义（移除了 Tailwind 样式，替换为了 `colorHex` 颜色值）。
   - `Engine.ts`: 核心算法引擎（包含网格碰撞检测、羁绊计算、属性加成等纯逻辑）。
   - `GameManager.ts`: 游戏主控制器（继承自 `Component`，负责管理金币、商店刷新、背包状态，并提供了 `tryPlaceItem` 等接口）。
   - `DraggableItem.ts`: 拖拽组件（继承自 `Component`，监听了 Cocos 的 `TOUCH_START/MOVE/END` 事件，处理物品的拖拽与吸附）。
3. **在 Cocos 中使用**：
   - 在场景中创建一个空节点命名为 `GameManager`，挂载 `GameManager.ts` 脚本。
   - 将 UI 节点（如金币 Label、网格容器 Node）拖拽绑定到 `GameManager` 的 `@property` 上。
   - 制作一个物品预制体（Prefab），挂载 `DraggableItem.ts` 脚本，并在 `GameManager` 中引用该预制体。

## 🚀 如何运行 (Web 版本)

1. 安装依赖:
   ```bash
   npm install
   ```
2. 启动开发服务器:
   ```bash
   npm run dev
   ```
3. 构建生产版本:
   ```bash
   npm run build
   ```
